Key Design Decisions

  • OCR is abstracted — includes/ocr.php exposes a single function: extract_fields($file_path) → returns associative array. Swap Tesseract for Google Vision by changing only this file.
  • Template-driven — The SO document format lives in a .docx template file, not hardcoded in PHP. Registrar office can update the template without developer help.
  • Database tracking — Each generated SO gets a record in school_orders for audit trail, SO number auto-increment, and re-download.
  • No new upload flow — Uses existing PDFs from request_files table.
