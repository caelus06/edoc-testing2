Design Spec: OCR-Driven School Order Generation

Review the full spec below. Let me know in the terminal if you want changes.

Overview

Allow registrar staff to select a verified student request, run Tesseract OCR on the student's uploaded PDF to extract key data, review/correct the extracted fields in a split-screen UI, and generate a formal "School Order" document as DOCX (with PDF download option).

Key Decisions

OCR Engine: Tesseract CLI (local), swappable to Google Vision later via includes/ocr.php
Source Files: Reuse existing uploaded PDFs from request_files table
Document Types: SO available for all document types (any verified request)
SO Format: Flexible .docx template, editable without code changes
SO Number: Auto-generated SO-{YEAR}-{4-digit}, editable by registrar
Export: DOCX via PHPWord (primary), PDF via LibreOffice CLI (optional)

Tesseract: PDF → images via Ghostscript → tesseract <image> stdout → regex parse fields
Multi-page: All pages OCR'd, text concatenated
Confidence: Exact regex = high, partial = medium, not found = low
Google Vision swap: Replace function body only, credentials via config constant

Security & Error Handling

Security
  • CSRF on all POST forms
  • require_role(ROLE_REGISTRAR)
  • escapeshellarg() on CLI args
  • Downloads via PHP readfile(), not direct URL
  • Audit logging on create/finalize/void
Error Handling
  • OCR fails → error banner, manual entry
  • PHPWord fails → error msg, no incomplete save
  • LibreOffice missing → hide PDF button
  • Duplicate SO# → validation error
  • Wrong status → request rejected
